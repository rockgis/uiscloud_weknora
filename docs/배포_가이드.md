# WeKnora 배포 가이드

GitHub Releases에서 공개된 이미지(ghcr.io)를 사용하여 서버에 WeKnora를 배포하는 방법입니다.
소스 코드 빌드 없이 Docker Compose만으로 운영 환경을 구성할 수 있습니다.

## 사전 요구사항

| 항목 | 최소 | 권장 |
|------|------|------|
| OS | Linux (Ubuntu 20.04+) / macOS | Ubuntu 22.04 LTS |
| CPU | 2 cores | 4 cores |
| RAM | 4 GB | 8 GB 이상 |
| 디스크 | 20 GB | 50 GB 이상 |
| Docker | 20.10+ | 최신 |
| Docker Compose | v2.0+ | 최신 |

### Docker 설치 (Ubuntu)

```bash
curl -fsSL https://get.docker.com | bash
sudo usermod -aG docker $USER
# 로그아웃 후 재로그인
```

## 원라인 설치

```bash
curl -fsSL https://github.com/rockgis/uiscloud_weknora/releases/latest/download/install.sh | bash
```

스크립트가 자동으로 다음 작업을 수행합니다:
1. Docker / Docker Compose 설치 여부 확인
2. 최신 릴리즈의 `docker-compose.yml`, `.env.example` 다운로드
3. `.env` 파일 생성 및 비밀번호 자동 생성 (DB, Redis, JWT, AES 키)
4. 설정 검토 안내 출력 → `vi ~/weknora/.env` 로 필수 항목 설정

> **설치 경로 변경**: `WEKNORA_DIR=/opt/weknora bash install.sh`

## 수동 설치

자동 스크립트 대신 직접 설치하는 방법입니다.

### 1. 릴리즈 파일 다운로드

```bash
VERSION="0.2.5"   # 설치할 버전으로 변경

mkdir -p ~/weknora && cd ~/weknora

curl -fsSL "https://github.com/rockgis/uiscloud_weknora/releases/download/v${VERSION}/docker-compose.yml" \
  -o docker-compose.yml

curl -fsSL "https://github.com/rockgis/uiscloud_weknora/releases/download/v${VERSION}/.env.example" \
  -o .env.example
```

### 2. .env 파일 설정

```bash
cp .env.example .env
vi .env
```

### 3. 서비스 시작

```bash
docker compose pull
docker compose up -d
```

## .env 주요 설정

`.env` 파일에서 반드시 검토해야 할 항목입니다.

### 보안 (필수 변경)

```env
# 랜덤 값으로 자동 생성되지만 확인 권장
DB_PASSWORD=<강력한-비밀번호>
REDIS_PASSWORD=<강력한-비밀번호>
JWT_SECRET=<32자-이상-랜덤-문자열>
TENANT_AES_KEY=<32자-암호화-키>

# 운영 모드 (프로덕션에서는 반드시 release)
GIN_MODE=release
```

### LLM 연결

```env
# Ollama 사용 시 (로컬 또는 원격)
OLLAMA_BASE_URL=http://host.docker.internal:11434

# 초기 생성될 LLM 모델 (Ollama에 pull 되어 있어야 함)
INIT_LLM_MODEL_NAME=qwen2.5:7b
INIT_LLM_CONTEXT_WINDOW=32768

# 임베딩 모델
INIT_EMBEDDING_MODEL_NAME=nomic-embed-text
INIT_EMBEDDING_DIMENSIONS=768
```

### 스토리지

```env
# local (기본), minio, cos 중 선택
STORAGE_TYPE=local
LOCAL_STORAGE_BASE_DIR=/data/files   # 컨테이너 내부 경로 (볼륨으로 연결됨)

# MinIO 사용 시 (STORAGE_TYPE=minio)
# MINIO_ENDPOINT=minio:9000
# MINIO_BUCKET_NAME=weknora
```

### 검색 엔진

```env
# postgres (기본, pgvector), elasticsearch_v8, qdrant 중 선택
RETRIEVE_DRIVER=postgres
```

### 포트 (충돌 시 변경)

```env
APP_PORT=8080
FRONTEND_PORT=80
```

## 서비스 관리

설치 디렉터리(`~/weknora`)에서 실행하세요.

```bash
cd ~/weknora

# 상태 확인
docker compose ps

# 실시간 로그 (전체)
docker compose logs -f

# 특정 서비스 로그
docker compose logs -f app
docker compose logs -f docreader

# 서비스 재시작
docker compose restart app

# 중지 (데이터 보존)
docker compose down

# 중지 + 볼륨 삭제 (데이터 초기화 ⚠)
docker compose down -v
```

### 서비스 구성

| 서비스 | 역할 | 포트 |
|--------|------|------|
| `frontend` | Vue.js 웹 UI (Nginx) | 80 |
| `app` | Go 백엔드 API | 8080 |
| `docreader` | 문서 파싱 gRPC 서비스 | 50051 (내부) |
| `postgres` | PostgreSQL + pgvector | 5432 (내부) |
| `redis` | 캐시 / 작업 큐 | 6379 (내부) |

### 접속 URL

| 항목 | URL |
|------|-----|
| 웹 UI | http://서버IP |
| 백엔드 API | http://서버IP:8080 |
| API 문서 | http://서버IP:8080/swagger/index.html |

## 업데이트

```bash
cd ~/weknora

# 새 버전의 docker-compose.yml 다운로드 (버전 변경 시)
VERSION="0.2.6"
curl -fsSL "https://github.com/rockgis/uiscloud_weknora/releases/download/v${VERSION}/docker-compose.yml" \
  -o docker-compose.yml

# 이미지 업데이트 및 재시작
docker compose pull
docker compose up -d

# 사용하지 않는 이미지 정리
docker image prune -f
```

또는 install.sh 로 최신 버전 재설치:

```bash
curl -fsSL https://github.com/rockgis/uiscloud_weknora/releases/latest/download/install.sh | bash
```

> 기존 `.env` 파일이 있으면 유지되고 이미지만 업데이트됩니다.

## 선택적 서비스

`docker-compose.yml` 의 프로파일을 활성화하여 추가 서비스를 사용할 수 있습니다.

```bash
# MinIO 파일 스토리지 추가
docker compose --profile minio up -d

# Qdrant 벡터 데이터베이스 추가
docker compose --profile qdrant up -d

# Neo4j 그래프 데이터베이스 추가
docker compose --profile neo4j up -d

# Jaeger 분산 트레이싱 추가
docker compose --profile jaeger up -d

# 전체 서비스
docker compose --profile full up -d
```

| 프로파일 | 서비스 | 접속 |
|----------|--------|------|
| `minio` | MinIO 오브젝트 스토리지 | http://서버IP:9001 |
| `qdrant` | Qdrant 벡터 DB | http://서버IP:6333 |
| `neo4j` | Neo4j 그래프 DB | http://서버IP:7474 |
| `jaeger` | Jaeger UI | http://서버IP:16686 |

## 데이터 백업

### PostgreSQL 백업

```bash
cd ~/weknora

# 덤프 생성
docker compose exec postgres pg_dump -U ${DB_USER:-postgres} ${DB_NAME:-uiscloud_weknora} \
  > backup-$(date +%Y%m%d).sql

# 복원
docker compose exec -T postgres psql -U ${DB_USER:-postgres} ${DB_NAME:-uiscloud_weknora} \
  < backup-20250101.sql
```

### 파일 볼륨 백업

```bash
# 업로드 파일 백업
docker run --rm \
  -v weknora_data-files:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/files-$(date +%Y%m%d).tar.gz /data
```

## 문제 해결

### 서비스가 시작되지 않음

```bash
# 전체 로그 확인
docker compose logs

# 특정 서비스 상세 로그
docker compose logs --tail=100 app
docker compose logs --tail=100 postgres
```

### 포트 충돌

```bash
# 사용 중인 포트 확인
sudo lsof -i :80
sudo lsof -i :8080

# .env에서 포트 변경
APP_PORT=18080
FRONTEND_PORT=18080
```

### 데이터베이스 연결 실패

```bash
# PostgreSQL 직접 접속 테스트
docker compose exec postgres psql -U postgres -d uiscloud_weknora -c '\l'

# 데이터베이스 초기화 (⚠ 데이터 삭제)
docker compose down -v
docker compose up -d
```

### 메모리 부족

```bash
# 현재 메모리 사용량
docker stats --no-stream

# 불필요한 이미지 정리
docker system prune -a
```

### Ollama 연결 실패

```bash
# 호스트에서 Ollama 실행 확인
curl http://localhost:11434/api/tags

# 컨테이너에서 호스트 접근 테스트
docker compose exec app curl http://host.docker.internal:11434/api/tags
```

> Linux에서 `host.docker.internal`이 동작하지 않는 경우 `.env`에서
> `OLLAMA_BASE_URL=http://172.17.0.1:11434` 로 변경하세요 (Docker 브리지 게이트웨이 IP).

## GitHub Container Registry 이미지

WeKnora의 Docker 이미지는 GitHub Container Registry에 공개되어 있습니다.

| 이미지 | 태그 | 설명 |
|--------|------|------|
| `ghcr.io/rockgis/weknora-ui` | `latest`, `vX.Y.Z` | Vue.js 프론트엔드 |
| `ghcr.io/rockgis/weknora-app` | `latest`, `vX.Y.Z` | Go 백엔드 API |
| `ghcr.io/rockgis/weknora-docreader` | `latest`, `vX.Y.Z` | Python 문서 파서 |

- `latest`: `main` 브랜치 최신 빌드
- `vX.Y.Z`: 특정 릴리즈 버전 (재현 가능한 배포에 권장)
