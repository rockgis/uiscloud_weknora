# 지식 그래프 기능 활성화 가이드

이 문서에서는 WeKnora에서 지식 그래프(Neo4j) 기능을 활성화하고 검증하는 방법을 설명합니다. 환경 준비부터 프론트엔드 구성까지 전체 프로세스를 안내합니다.

## 사전 요구 사항

- WeKnora 백엔드 및 프론트엔드의 기본 배포가 완료되어야 합니다.
- Docker/Docker Compose 실행 환경이 사용 가능해야 합니다.
- 로컬 또는 원격에서 접근 가능한 Neo4j 서비스가 필요합니다 (프로젝트에 포함된 Docker Compose 사용 권장).

## 1단계: 환경 변수 구성

프로젝트 루트 디렉토리의 `.env` 파일에 다음 변수를 추가하거나 수정하세요:

```
NEO4J_ENABLE=true
NEO4J_URI=bolt://neo4j:7687
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=your_strong_password
# 선택사항: NEO4J_DATABASE=neo4j
```

설명:

- `NEO4J_ENABLE`을 `true`로 설정해야 지식 그래프 관련 로직이 활성화됩니다.
- `NEO4J_URI`의 `neo4j`는 docker-compose 서비스 이름입니다. 외부 인스턴스를 사용하는 경우 실제 주소로 교체하세요.
- 프로덕션 환경에서 비밀 관리를 사용하는 경우, 비밀번호가 안전한 방식으로 주입되어야 합니다.

## 2단계: Neo4j 서비스 시작

프로젝트에 Neo4j 컴포넌트가 포함되어 있으며, 다음 명령으로 직접 시작할 수 있습니다:

```bash
docker-compose --profile neo4j up -d
```

일반적인 확인 명령:

```bash
docker ps | grep neo4j
```

마운트 또는 메모리를 사용자 정의해야 하는 경우, `docker-compose.yml`의 `neo4j` 서비스 구성을 편집하세요.

## 3단계: WeKnora 서비스 재시작

새 환경 변수를 적용하려면 백엔드와 프론트엔드를 재시작하세요 (예시 참조):

```bash
make stop && make start
# 또는
docker compose up -d --build
```

백엔드 로그에서 `neo4j` 초기화 성공 메시지가 나타나는지 확인하세요.

## 4단계: 프론트엔드에서 엔티티/관계 추출 활성화

1. WeKnora 프론트엔드 관리 페이지에 로그인합니다.
2. "지식베이스 설정"을 열거나 새 지식베이스를 생성합니다.
3. "엔티티 추출 활성화" 및 "관계 추출 활성화" 스위치를 켭니다.
4. 화면 안내에 따라 필요한 LLM, 콜백 또는 모델 파라미터를 입력합니다 (필요한 경우).

저장 후, 시스템은 문서 인덱싱 단계에서 자동으로 엔티티 및 관계 추출 작업을 트리거합니다.

## 5단계: 지식 그래프 검증

### 방법 1: Neo4j 콘솔

1. `http://localhost:7474` (또는 해당 호스트/포트)에 접속합니다.
2. `.env`의 계정과 비밀번호로 로그인합니다.
3. `MATCH (n) RETURN n LIMIT 50;`을 실행하여 새 노드/관계가 있는지 확인합니다.

### 방법 2: WeKnora 인터페이스

지식베이스 또는 대화 페이지에서 문서를 업로드한 후, 프론트엔드에 그래프 시각화 진입점이 표시되어야 합니다. 대화 중 시스템은 의도에 따라 자동으로 그래프를 쿼리하고 보충 정보를 반환합니다.

## 일반적인 문제 해결

- **Neo4j에 연결할 수 없음**: 네트워크 연결 가능 여부, `NEO4J_URI` 및 사용자 이름/비밀번호가 정확한지 확인하고, Neo4j 컨테이너 로그를 확인하세요.
- **노드가 생성되지 않음**: 지식베이스에서 엔티티/관계 추출이 활성화되어 있고, 업로드된 문서가 파싱 완료되었는지 확인하세요. 백엔드 로그에서 추출 작업 오류가 있는지 확인하세요.
- **쿼리 결과 없음**: Neo4j 콘솔에서 `CALL db.schema.visualization;`을 실행하여 스키마가 존재하는지 확인하고, 필요한 경우 문서를 다시 가져오세요.

위 단계를 완료하면 지식 그래프 기능이 성공적으로 활성화됩니다. RAG 및 Agent 플로우와 결합하여 Q&A 품질을 향상시킬 수 있습니다.
