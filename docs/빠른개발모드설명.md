# 빠른 개발 모드 설명

개발 과정에서 `app`(백엔드) 또는 `frontend`(프론트엔드) 코드를 수정할 때마다 Docker 이미지를 다시 빌드해야 하는 문제를 해결하고, 이 두 모듈의 핫 리로드를 구현합니다.


## 🚀 사용 방법

### 방법 1: Make 명령 사용 (권장)

```bash
# 터미널 1: 인프라 시작
make dev-start

# 터미널 2: 백엔드 시작
make dev-app

# 터미널 3: 프론트엔드 시작
make dev-frontend
```

### 방법 2: 개발 스크립트 사용

```bash
# 터미널 1
./scripts/dev.sh start

# 터미널 2
./scripts/dev.sh app

# 터미널 3
./scripts/dev.sh frontend
```

### 방법 3: 원클릭 시작 (대화형)

```bash
./scripts/quick-dev.sh
```



### Air를 사용한 백엔드 핫 리로드

Air를 설치하면 백엔드 코드 수정 시 자동으로 다시 컴파일되고 재시작됩니다:

```bash
# Air 설치
go install github.com/cosmtrek/air@latest

# PATH에 추가 확인
export PATH=$PATH:$(go env GOPATH)/bin

# Air로 시작 (자동 감지)
make dev-app
```
