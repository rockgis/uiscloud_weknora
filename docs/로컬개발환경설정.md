## 빠른 시작

### 1️⃣ 환경 변수 설정

```bash
# .env 파일 생성
cp .env.example .env
```

`.env` 파일에서 필수 값을 설정하세요:

```bash
# 데이터베이스
DB_USER=postgres
DB_PASSWORD=postgres123!@#
DB_NAME=WeKnora

# Redis
REDIS_PASSWORD=redis123!@#

# 보안
JWT_SECRET=weknora-jwt-secret
TENANT_AES_KEY=weknorarag-api-key-secret-secret

# 스토리지
STORAGE_TYPE=local
LOCAL_STORAGE_BASE_DIR=./data/files

# 벡터 검색
RETRIEVE_DRIVER=postgres

# 스트림 매니저
STREAM_MANAGER_TYPE=redis

# 모드
GIN_MODE=debug
```

### 2️⃣ 인프라 서비스 시작

```bash
# 기본 서비스만 시작 (PostgreSQL + Redis + DocReader)
make dev-start
```

**또는 추가 서비스 포함:**

```bash
# Qdrant 벡터 DB 포함
make dev-start --qdrant

# Qdrant + Jaeger 추적 포함
make dev-start --qdrant --jaeger

# 모든 서비스 포함 (MinIO, Qdrant, Neo4j, Jaeger)
make dev-start --full
```

시작 후 다음과 같은 메시지가 표시됩니다:

```
[SUCCESS] 인프라 서비스가 시작되었습니다

[INFO] 서비스 접속 주소:
  - PostgreSQL:    localhost:5432
  - Redis:         localhost:6379
  - DocReader:     localhost:50051

[INFO] 다음 단계:
1. 새 터미널에서 백엔드 실행: make dev-app
2. 새 터미널에서 프론트엔드 실행: make dev-frontend
```

### 3️⃣ 애플리케이션 실행

**터미널 1 - 백엔드:**

```bash
make dev-app
```

Air가 설치된 경우 핫 리로드가 활성화됩니다. Go 파일 수정 시 자동으로 재시작됩니다.

**터미널 2 - 프론트엔드:**

```bash
make dev-frontend
```

Vite 개발 서버가 시작되며, 파일 수정 시 자동으로 핫 리로드됩니다.

### 4️⃣ 접속

- **프론트엔드**: http://localhost:5173
- **백엔드 API**: http://localhost:8080
- **Swagger 문서**: http://localhost:8080/swagger/index.html

## 상세 명령어

### 개발 환경 관리

#### 서비스 시작

```bash
# 기본 서비스만 (PostgreSQL + Redis + DocReader)
make dev-start

# Qdrant 벡터 DB 추가
make dev-start --qdrant

# 여러 서비스 조합
make dev-start --qdrant --neo4j
make dev-start --qdrant --jaeger

# 모든 선택 서비스 포함
make dev-start --full
```

#### 서비스 상태 확인

```bash
# 실행 중인 컨테이너 목록
make dev-status

# 또는
./scripts/dev.sh status
```

출력 예시:
```
NAME                      IMAGE                             STATUS
WeKnora-postgres-dev      paradedb/paradedb:v0.18.9-pg17   Up 5 minutes (healthy)
WeKnora-redis-dev         redis:7.0-alpine                 Up 5 minutes
WeKnora-docreader-dev     wechatopenai/weknora-docreader   Up 5 minutes (healthy)
```

#### 로그 확인

```bash
# 모든 서비스 로그 (실시간)
make dev-logs

# 또는
./scripts/dev.sh logs
```

#### 서비스 재시작

```bash
make dev-restart

# 또는
./scripts/dev.sh restart
```

#### 서비스 중지

```bash
make dev-stop

# 또는
./scripts/dev.sh stop
```

### 애플리케이션 실행

#### 백엔드

```bash
# 핫 리로드 지원 (Air 설치 시)
make dev-app

# 또는
./scripts/dev.sh app
```

Air가 없는 경우:
```bash
# Air 설치 권장
go install github.com/air-verse/air@latest

# Air 없이 실행 (수동 재시작 필요)
go run cmd/server/main.go
```

#### 프론트엔드

```bash
# Vite 개발 서버 (핫 리로드 지원)
make dev-frontend

# 또는
./scripts/dev.sh frontend
```

처음 실행 시 의존성을 자동으로 설치합니다:
```bash
cd frontend
pnpm install
pnpm dev
```

## 환경 변수 설정

### 필수 설정

#### 데이터베이스 설정

```bash
# 데이터베이스 드라이버 (postgres/mysql)
DB_DRIVER=postgres

# 데이터베이스 사용자명
DB_USER=postgres

# 데이터베이스 비밀번호
DB_PASSWORD=postgres123!@#

# 데이터베이스 이름
DB_NAME=WeKnora

# 포트 (선택, 기본값 5432)
# DB_PORT=5432
```

#### Redis 설정

```bash
# Redis 비밀번호
REDIS_PASSWORD=redis123!@#

# Redis 데이터베이스 인덱스 (0-15)
REDIS_DB=0

# Redis 키 접두사 (네임스페이스 격리)
REDIS_PREFIX=stream:

# 포트 (선택, 기본값 6379)
# REDIS_PORT=6379
```

#### 보안 설정

```bash
# JWT 토큰 시크릿 (최소 32자 권장)
JWT_SECRET=weknora-jwt-secret

# 테넌트 API 키 암호화 키 (32자)
TENANT_AES_KEY=weknorarag-api-key-secret-secret
```

#### 스토리지 설정

```bash
# 파일 스토리지 타입 (local/minio/cos)
STORAGE_TYPE=local

# 로컬 스토리지 기본 디렉토리
LOCAL_STORAGE_BASE_DIR=./data/files

# 더티 데이터 자동 복구 여부
AUTO_RECOVER_DIRTY=true
```

#### 검색 엔진 설정

```bash
# 벡터 검색 드라이버 (postgres/elasticsearch_v7/elasticsearch_v8/qdrant)
RETRIEVE_DRIVER=postgres
```

#### 스트림 매니저 설정

```bash
# 스트림 처리 백엔드 (memory/redis)
STREAM_MANAGER_TYPE=redis
```

#### 애플리케이션 설정

```bash
# Gin 모드 (debug/release)
GIN_MODE=debug

# 애플리케이션 포트
APP_PORT=8080

# 프론트엔드 포트
FRONTEND_PORT=80

# 문서 파서 포트
DOCREADER_PORT=50051

# 동시 처리 수 (429 오류 발생 시 줄이기)
CONCURRENCY_POOL_SIZE=5
```

### 선택 설정

#### Ollama 연동 (로컬 LLM)

```bash
# Ollama 서비스 기본 URL
OLLAMA_BASE_URL=http://host.docker.internal:11434

# 초기 LLM 모델 설정

INIT_EMBEDDING_MODEL_BASE_URL=http://localhost:11434
# INIT_EMBEDDING_MODEL_ID=  # 선택

# 초기 재순위 모델 설정
INIT_RERANK_MODEL_NAME=bge-reranker-v2-m3
INIT_RERANK_MODEL_BASE_URL=http://localhost:11434
# INIT_RERANK_MODEL_API_KEY=  # Ollama는 API 키 불필요
```

#### MinIO 설정

```bash
# 스토리지 타입을 MinIO로 변경
```

## 개발 워크플로우

### 시나리오 1: 기본 개발 (PostgreSQL + 로컬 스토리지)

```bash
# 1. .env 파일 설정
cp .env.example .env
vim .env  # 필수 값 설정

# 2. 인프라 시작
make dev-start

# 3. 터미널 1 - 백엔드
make dev-app

# 4. 터미널 2 - 프론트엔드
make dev-frontend

# 5. 브라우저 접속
# http://localhost:5173
```

**환경 변수:**
```bash
DB_DRIVER=postgres
RETRIEVE_DRIVER=postgres
STORAGE_TYPE=local
STREAM_MANAGER_TYPE=redis
```

### 시나리오 2: Qdrant 벡터 검색 개발

```bash
# 1. Qdrant 포함하여 인프라 시작
make dev-start --qdrant

# 2. .env 파일 수정
vim .env
# RETRIEVE_DRIVER=qdrant

# 3. 백엔드/프론트엔드 시작
make dev-app        # 터미널 1
make dev-frontend   # 터미널 2

# 4. Qdrant 대시보드 확인
# http://localhost:6333/dashboard
```

**환경 변수:**
```bash
RETRIEVE_DRIVER=qdrant
QDRANT_HOST=localhost
QDRANT_PORT=6334
QDRANT_COLLECTION=weknora_embeddings
```

### 시나리오 3: 그래프 RAG 개발 (Neo4j)

```bash
# 1. Neo4j 포함하여 인프라 시작
make dev-start --neo4j

# 2. .env 파일 수정
vim .env
# ENABLE_GRAPH_RAG=true
# NEO4J_ENABLE=true

# 3. 백엔드/프론트엔드 시작
make dev-app        # 터미널 1
make dev-frontend   # 터미널 2

# 4. Neo4j 브라우저 접속
# http://localhost:7474
# 사용자명: neo4j
# 비밀번호: password
```

**환경 변수:**
```bash
ENABLE_GRAPH_RAG=true
NEO4J_ENABLE=true
NEO4J_URI=bolt://localhost:7687
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=password
```

### 시나리오 4: 전체 기능 개발 (분산 추적 포함)

```bash
# 1. 모든 서비스 시작
make dev-start --full

# 2. .env 파일에서 필요한 서비스 활성화
vim .env

# 3. 백엔드/프론트엔드 시작
make dev-app        # 터미널 1
make dev-frontend   # 터미널 2

# 4. 서비스 접속
# - Jaeger UI: http://localhost:16686
# - MinIO Console: http://localhost:9001
# - Neo4j Browser: http://localhost:7474
# - Qdrant Dashboard: http://localhost:6333/dashboard
```

**환경 변수:**

```bash
STORAGE_TYPE=minio
```

## 트러블슈팅

### 포트 충돌

**문제:** 포트가 이미 사용 중입니다.

```
Error: bind: address already in use
```

**해결 방법:** `.env` 파일에서 포트 변경

```bash
# PostgreSQL 포트 변경
DB_PORT=5433

# Redis 포트 변경
REDIS_PORT=6380

# DocReader 포트 변경
DOCREADER_PORT=50052

# MinIO 포트 변경
MINIO_PORT=9002
MINIO_CONSOLE_PORT=9003
```

**사용 중인 포트 확인:**

```bash
# macOS/Linux
lsof -i :5432
lsof -i :6379

# 프로세스 종료
kill -9 <PID>
```

### Docker 권한 문제

**문제:** Docker 명령어 실행 시 권한 오류

```
permission denied while trying to connect to the Docker daemon socket
```

**해결 방법:**

```bash
# Docker 데몬 상태 확인
docker info

# Docker Desktop 재시작 (macOS/Windows)
# 또는 Docker 서비스 재시작 (Linux)
sudo systemctl restart docker

# 사용자를 docker 그룹에 추가 (Linux)
sudo usermod -aG docker $USER
newgrp docker
```

### Docker Compose 버전 문제

**문제:** Docker Compose 명령어가 인식되지 않습니다.

```
docker-compose: command not found
```

**해결 방법:**

```bash
# Docker Compose v2 확인
docker compose version

# Docker Compose v1 확인
docker-compose version

# 스크립트는 자동으로 감지하지만, 수동 설치 필요 시:
# https://docs.docker.com/compose/install/
```

### 데이터베이스 연결 실패

**문제:** 백엔드가 데이터베이스에 연결할 수 없습니다.

```
failed to connect to database: connection refused
```

**해결 방법:**

1. PostgreSQL 컨테이너 상태 확인:
```bash
make dev-status
docker logs WeKnora-postgres-dev
```

2. 헬스 체크 확인:
```bash
docker exec WeKnora-postgres-dev pg_isready -U postgres
```

3. .env 파일 확인:
```bash
DB_HOST=localhost  # Docker 컨테이너 외부에서 접속 시
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=postgres123!@#
DB_NAME=WeKnora
```

4. 데이터베이스 재시작:
```bash
make dev-stop
make dev-start
```

### 데이터베이스 초기화

**문제:** 기존 데이터를 삭제하고 새로 시작하고 싶습니다.

**해결 방법:**

```bash
# 1. 서비스 중지
make dev-stop

# 2. Docker 볼륨 삭제
docker volume rm weknora_postgres-data-dev
docker volume rm weknora_redis_data_dev

# 3. 서비스 재시작
make dev-start
```

### DocReader gRPC 연결 실패

**문제:** 문서 파싱 시 DocReader에 연결할 수 없습니다.

```
failed to connect to docreader: connection refused
```

**해결 방법:**

1. DocReader 컨테이너 상태 확인:
```bash
docker logs WeKnora-docreader-dev
```

2. gRPC 헬스 체크:
```bash
docker exec WeKnora-docreader-dev grpc_health_probe -addr=:50051
```

```bash
DOCREADER_ADDR=localhost:50051
```

### Ollama 연결 실패

**문제:** Ollama 서비스에 연결할 수 없습니다.

```
failed to connect to ollama: connection refused
```

**해결 방법:**

1. Ollama 서비스 상태 확인:

```bash
curl http://localhost:11434/api/tags
```

2. Ollama 재시작:
```bash
# macOS/Linux
brew services restart ollama

# 또는 수동 실행
ollama serve
```

3. .env 파일 확인:
```bash
# Docker 내부에서 호스트 접속
OLLAMA_BASE_URL=http://host.docker.internal:11434

# 로컬 개발 시
OLLAMA_BASE_URL=http://localhost:11434
```

### 메모리 부족

**문제:** Docker 컨테이너 실행 시 메모리 부족 오류

**해결 방법:**

1. Docker Desktop 설정에서 메모리 할당 증가:
   - macOS: Docker Desktop → Preferences → Resources → Memory (8GB 이상 권장)
   - Windows: Docker Desktop → Settings → Resources → Memory

2. 불필요한 서비스 비활성화:
```bash
# 전체 서비스 대신 필요한 것만 시작
make dev-start  # 기본 서비스만
```

**유용한 psql 명령어:**
```sql
-- 테이블 목록

-- 쿼리 실행
SELECT * FROM knowledge_bases LIMIT 10;

-- pgvector 확인
SELECT * FROM pg_extension WHERE extname = 'vector';

-- 종료
\q
```

- Database: `WeKnora`
- Username: `postgres`
- Password: `.env`에 설정한 `DB_PASSWORD`
